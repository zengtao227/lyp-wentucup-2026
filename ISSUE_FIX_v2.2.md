# v2.2 紧急修复报告

## 🚨 **严重问题：Agent 输出工具调用标记而不是执行工具**

### 问题现象

**用户输入**：
```
5月20号还有豪华大床房吗
```

**Agent 错误输出**：
```
<｜tool▁calls▁begin｜><｜tool▁call▁begin｜>function<｜tool▁sep｜>getAvailableRoomsByRoomTypeAndDate
{"teamId": 234556, "roomType": "002", "checkInDate": "2026-05-20", "checkOutDate": "2026-05-21"}<｜tool▁call▁end｜><｜tool▁calls▁end｜>
```

**期望输出**：
```
让我为您查询一下5月20日豪华大床房的房态...
[内部调用工具]
好的，5月20日豪华大床房有空房，价格 ¥500/晚，剩余 135 间...
```

---

## 🔍 **根本原因分析**

### 原因 1：Temperature 设置过低

```yaml
# v2.1 配置（错误）
temperature: 0.1  ❌ 过低，导致模型行为异常
```

**影响**：
- Temperature 过低会让模型过于保守
- DeepSeek-V3 在 Agent 模式下，低 temperature 可能导致输出格式错误
- 模型可能把工具调用标记当成普通文本输出

### 原因 2：提示词约束不够强

```yaml
# v2.1 提示词
- 严禁输出任何 Thought, Action, Observation, Final Answer 等技术标签
```

**问题**：
- 只禁止了 ReAct 的标签，没有禁止工具调用标记
- 没有明确说明"你是客服，不是代码生成器"
- 缺少正面引导（应该怎么做）

---

## ✅ **v2.2 修复方案**

### 修复 1：调整 Temperature

```yaml
# v2.2 配置（修复）
temperature: 0.7  ✅ 提高到 0.7，增加模型灵活性
```

**理由**：
- 0.7 是 Agent 模式的推荐值
- 既保持稳定性，又有足够的灵活性
- 符合 DeepSeek-V3 的最佳实践

### 修复 2：增强提示词约束

```yaml
# v2.2 新增核心规则（最高优先级）

你是一个智能客服助手，NOT a code generator or API documentation tool.

**绝对禁止**：
- 禁止输出任何工具调用标记（如 <｜tool▁calls▁begin｜>、function、tool_sep 等）
- 禁止输出任何技术标签（Thought、Action、Observation、Final Answer）
- 禁止输出任何 JSON 格式的原始数据
- 禁止输出任何代码片段或 API 调用示例

**必须做到**：
- 使用自然、流畅的中文与客人对话
- 内部调用工具获取信息，但只向客人展示最终结果
- 像真人客服一样礼貌、专业、热情
```

**改进点**：
1. ✅ 明确身份定位："智能客服助手，NOT a code generator"
2. ✅ 具体列出禁止输出的标记（包括工具调用标记）
3. ✅ 正面引导：应该怎么做（自然对话、展示结果）
4. ✅ 放在提示词最开头（最高优先级）

## 📝 **技术总结**

### 经验教训

1. **Temperature 很重要**
   - Agent 模式推荐 0.7
   - 不要盲目追求低 temperature

2. **提示词要具体**
   - 不仅要说"禁止"，还要说"为什么"
   - 正面引导比负面约束更有效

3. **模型有个性**
   - DeepSeek-V3 在 Agent 模式下需要特殊配置
   - 不同模型的最佳实践不同

### 最佳实践

1. **提示词结构**
   ```
   核心规则（最高优先级）
   ↓
   工具调用规范
   ↓
   业务流程
   ↓
   具体任务
   ```

2. **Temperature 选择**
   ```
   分类任务：0.1-0.3
   问答任务：0.5-0.7
   Agent 任务：0.7-0.9
   创意任务：0.9-1.0
   ```

3. **测试策略**
   ```
   基础功能 → 边界情况 → 完整流程
   ```
