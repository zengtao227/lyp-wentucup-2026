### 4. 测试对话

在预览窗口测试以下场景：

**场景 1：正常订房**
```
用户：我想订5月20号的高级大床房
Agent：好的！请问您计划什么时候离店？
```

## 📊 技术架构

### Agent 模式

```
用户输入
    ↓
DeepSeek-V3 模型（temperature: 0.1）
    ↓
ReAct 策略（最大迭代 3 次）
    ↓
工具调用决策
    ↓
┌─────────────┬─────────────┬─────────────┐
│  MCP 工具   │  知识库检索  │  时间/天气   │
│  (14个)     │  (5个数据集) │  (5个工具)   │
└─────────────┴─────────────┴─────────────┘
    ↓
结果整合
    ↓
自然语言回复
```

### 房型代码转换流程

```
用户说："我想订标准间"
    ↓
Agent 识别：不在房型对照表中
    ↓
智能引导："我们没有标准间，请问您需要大床房还是双床房？"
    ↓
用户说："大床房"
    ↓
Agent 推荐："高级大床房（入门级）、豪华大床房（升级版）、行政大床房（高端）"
    ↓
用户选择："高级大床房"
    ↓
Agent 转换：高级大床房 → 房型代码 "001"
    ↓
调用 MCP 工具：getAvailableRoomsByRoomTypeAndDate(roomType="001", ...)
    ↓
返回房态数据
```

## 🎓 答辩要点

### 技术创新点

1. **智能引导机制**：不是简单拒绝，而是主动引导客人选择合适房型
2. **房型代码自动转换**：解决了中文名称与系统代码不匹配的问题
3. **防死循环设计**：避免工具重复调用，提升用户体验
