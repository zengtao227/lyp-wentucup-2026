④ 景区门票价格的独立算术节点

这是什么

当前景区购票的价格计算是由 LLM 自己心算：prompt 里写了「成人票 88 元，儿童票 44 元」，让 AI 去算 3×88+1×44=?

问题：大模型算数偶尔会出错。你希望评委对价格数据有信心，就不应该让 AI 心算。

解决方案

在 Dify 工作流中加一个 Code 节点（代码执行节点），固定公式：

总价 = 成人票数 × 成人单价 + 儿童票数 × 儿童单价

代码节点是确定的——永远算不错。

是否需要你做

不用你动手，我来改 YAML。 Dify 本身就提供 Code 节点功能。

当前加了算术节点后
流程 票务查询 → LLM（在prompt里算价格）→ 确认摘要 票务查询 → LLM → Code 节点算总价 → 确认摘要
价格准确率 大模型心算，可能出错 代码计算，100% 准确

技术方案说明（PPT 建议文案）：

Agent 通过 MCP（Model Context Protocol）协议直连酒店 PMS 与景区票务系统，实现从查房态 → 预订提交 → 取消退款的真实业务闭环：

  • 酒店客房：使用平台提供的 mcp-server-booking 服务，读取酒店后台实时房态与价格，提交的预订订单进入酒店预订系统

  • 取消/退款：通过 MCP 调用酒店取消预订接口，支持多轮确认流程

为什么选择 MCP 而非直接 HTTP 调用：

  • MCP 是标准化协议，Dify 原生支持工具节点调用
  • 无需处理 PMS 认证、鉴权、数据格式转换等底层细节
  • 平台内网通信，路演现场稳定可靠
