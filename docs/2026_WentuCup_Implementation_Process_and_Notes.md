# 🏆 2026 问途杯决赛：项目实现流程与注意事项

## 📌 核心技术范式：Chatflow vs. Agent

在问途 AI Playground 平台中，实现智能预订有两条完全不同的技术路径：

### 1. Agent 模式 (自主推理范式)
- **定义**：具备推理与自主工具调用的智能助手。
- **角色比喻**：**“主管制”**。你给它工具箱、业务 SOP 和边界，让模型自己判断“先查什么、再问什么”。
- **适用场景**：需求多变、希望对话自然、工具组合较灵活。
- **项目状态**：目前 `v2.0-Agent极简版` 采用此路径。

### 2. Chatflow 模式 (显式编排范式)
- **定义**：基于工作流编排，适用于复杂流程的多轮对话。
- **角色比喻**：**“导演制”**。你事先写好剧本，明确规定：节点 A 后必须跳到节点 B，如果不满足条件则跳到节点 C。
- **适用场景**：规则极其固定、需要强控每一步逻辑（如强制先收手机号再查房态）、追求极高的演示稳定性。

### 📊 路径对比表
| 特性 | Agent (自主推理) | Chatflow (显式编排) |
| --- | --- | --- |
| **控制力** | 软控制（靠 Prompt 指引） | 硬控制（靠节点连线） |
| **自然度** | 高（像真人，可随时被打断） | 一般（偏程式化，跳转较死板） |
| **开发难度** | 低（核心在写好 System Prompt） | 高（需要设计复杂的逻辑图） |
| **决赛策略建议** | **追求“智能化”表现的首选** | **如果 Agent 逻辑失控时的保底方案** |
